# Maratona de Programação

O objetivo da maratona de programação é construir algoritmos para resolver diversos problemas de programação. Confira abaixo as regras desta atividade:

1. Os problemas serão resolvidos por times de, no máximo, 3 (três) pessoas.
2. Os algoritmos deverão ser escritos na linguagem C.
3. Cada time terá 2 (duas) horas para tentar resolver o máximo de problemas que conseguir.
4. Para construir algoritmos, os times terão à disposição apenas um computador, e poderão utilizar todo material impresso à disposição.
5. É vetado aos times o acesso à internet.
6. A cada 20 (vinte) minutos, o time deverá substituir o integrante a manusear o computador.
7. Um problema é considerado resolvido se, para todos os testes, ele devolve o resultado esperado pelos juiz.
8. O time vencedor é aquele que resolve a maior quantidade de problemas nas 2 (duas) horas de competição.
9. Empates no número de problemas resolvidos são classificados pelo tempo corrigido. Persistindo o empate, passará à penúltima submissão correta e assim por diante. Se persistir o empate a organização fará um sorteio entre os times envolvidos.

Quando um time julgar que tem um programa que resolve um problema, ele pode submetê-lo à correção do juiz, que compila e executa este programa para uma bateria de testes desconhecida dos times.

Para submeter a possível solução de um problema, o time deverá acessar o sistema, através do endereço informado pelo juiz. Antes de submeter, o time deve informar usuário e senha adequados, e então acessar a tela de envio do arquivo.

O time deve certificar que a mensagem “Problema submetido com sucesso!” foi exibida, caso contrário, o problema não foi submetido para avaliação. Caso ocorram problemas na submissão, o time deve comunicar ao juiz imediatamente.

**BOA SORTE!**


## Problemas da maratona


1. Faça um algoritmo que leia um vetor `K[20]`. Troque, a seguir, todos os elementos de ordem ímpar do vetor com os elementos de ordem par imediatamente posteriores.

2. Preencha um vetor de 8 elementos inteiros. Mostre o vetor e informe quantos números são maiores que 30 e some esses números. Some também todos os números.

3. Em um vetor de 20 posições, preencha aleatoriamente as 10 primeiras posições com valores negativos e as 10 últimas com valores positivos.

4. Preencha um vetor de 30 posições com os valores de 10 a 20 e, depois, mostre os elementos pares do vetor de trás para frente.

5. Faça um algoritmo que verifique se uma matriz de ordem 3 é simétrica.

6. Faça um algoritmo que leia uma matriz `M[5][7]` e a exiba. Depois, verifique qual é o menor elemento e divida a matriz por ele. Exiba novamente a matriz.

7. Escreva um algoritmo que leia um vetor `G(13)`, que é o gabarito de um teste de loteria esportiva (números de 1 a 60). Leia, a seguir, para cada apostador, o número de seu cartão e um vetor resposta `R(13)`. Verifique o número de acertos e escreva o número do apostador e seu número de acertos. Se tiver 13 acertos, acrescente a mensagem: **"GANHADOR, PARABÉNS"**.

8. Crie uma função que receba 3 valores `float` e retorne o quadrado do primeiro mais a soma dos outros dois. Faça um `main()` que execute essa função.
