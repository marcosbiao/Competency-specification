# Título da Atividade

Sistema de gerenciamento de estoque e vendas de uma loja

# Contexto Instrucional

Atividade prática de programação em linguagem C, adequada a uma disciplina introdutória de programação estruturada. Pode ser desenvolvida individualmente ou em grupo, conforme a organização definida pela turma, em ambiente de laboratório, estudo dirigido ou avaliação prática com entrega de código-fonte e relatório técnico.

# Objetivo da Atividade

Desenvolver um programa em C que represente, de forma organizada e verificável, o funcionamento básico de um sistema de loja. A atividade busca integrar cadastro de dados, controle de estoque, registro de vendas, uso de estruturas de decisão e repetição, manipulação de vetores e apresentação de relatórios.

# Descrição da Tarefa

O estudante deve implementar um sistema em linguagem C para gerenciar uma loja fictícia, com identificação do empreendimento, dos produtos vendidos e do próprio sistema. O programa deve oferecer um menu principal com opções para cadastrar produtos, visualizar produtos cadastrados, registrar vendas, listar vendas e obter um relatório de desempenho.

Na rotina de cadastro de produtos, o sistema deve registrar informações como nome, preço de custo, preço de venda e quantidade em estoque, respeitando o limite máximo de produtos definido no enunciado. Na visualização de produtos, deve apresentar os itens cadastrados com seus respectivos estoques e preços, mantendo as quantidades atualizadas após as vendas.

Na rotina de vendas, o sistema deve permitir o registro de pedidos contendo um ou mais produtos, considerando as quantidades solicitadas e exibindo o valor total de cada pedido. Também deve armazenar as vendas realizadas até o limite indicado no enunciado e permitir sua listagem posterior, incluindo código da venda, quantidade de itens e valor correspondente.

O relatório de desempenho deve apresentar indicadores do negócio com base nos produtos cadastrados e nas vendas realizadas, incluindo arrecadação bruta, lucro, produto mais vendido, produto menos vendido, estoque remanescente desses produtos e valor médio das vendas. Além do programa, o estudante deve produzir um relatório técnico no formato solicitado pelo enunciado, descrevendo o processo de desenvolvimento e os resultados obtidos.

# Condições de Execução

* O programa deve ser desenvolvido em linguagem C.
* O sistema deve apresentar menu principal e permitir retorno às opções iniciais a partir das demais telas.
* O cadastro deve respeitar os limites máximos de produtos e vendas definidos no enunciado.
* O estoque deve ser atualizado de forma consistente após o registro de vendas.
* Cada venda deve conter informações suficientes para posterior listagem e composição dos indicadores do relatório.
* A solução deve utilizar estruturas de repetição, estruturas condicionais, múltiplos vetores e tipos de dados adequados aos diferentes dados manipulados.
* O programa deve tratar de forma coerente situações como ausência de produtos cadastrados, ausência de vendas registradas e tentativa de registrar quantidades incompatíveis com o estoque disponível.
* O relatório técnico deve descrever o processo de desenvolvimento, os resultados obtidos e a relação entre o comportamento do programa e os requisitos do enunciado.

# Evidências Esperadas

* Código-fonte em C no formato original, contendo a implementação funcional do sistema.
* Menu principal com as opções solicitadas e navegação coerente entre as funcionalidades.
* Rotina de cadastro e listagem de produtos com estoque atualizado.
* Rotina de registro e listagem de vendas com valores e quantidades consistentes.
* Relatório de desempenho produzido pelo programa com os indicadores exigidos.
* Uso verificável de estruturas de repetição, estruturas condicionais, vetores e tipos de dados adequados.
* Relatório técnico em PDF, no formato solicitado, descrevendo o desenvolvimento e os resultados do programa.

# Critérios de Avaliação

* Atendimento aos requisitos funcionais de cadastro, listagem, venda e relatório de desempenho.
* Correção do controle de estoque, do registro de vendas e dos indicadores apresentados.
* Uso adequado de estruturas de repetição, condicionais, vetores e tipos de dados em linguagem C.
* Completude, organização e verificabilidade do código-fonte e das saídas produzidas.
* Clareza técnica do relatório, incluindo descrição do processo de desenvolvimento e evidências dos resultados obtidos.
